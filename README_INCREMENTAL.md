# 증분 OHLCV 데이터 수집 (1997년부터)

`kospi_code.mst` 파일 기반으로 **1997년부터 현재까지**의 OHLCV 데이터를 증분 방식으로 수집합니다.

## 주요 특징

1. **1997년부터 시작**: 모든 종목의 1997년 1월 1일부터 현재까지 데이터 수집
2. **증분 수집**: 기존 데이터가 있으면 마지막 날짜 이후부터만 가져옵니다
3. **자동 병합**: 새로 가져온 데이터를 기존 데이터와 자동으로 병합
4. **중단 후 재개**: 중간에 중단되어도 다시 실행하면 이어서 가져옵니다
5. **API 제한 고려**: API 호출 제한을 피하기 위해 배치로 나눠서 가져옵니다

## 사용 방법

### 기본 실행

```bash
python get_ohlcv_incremental.py
```

### 옵션 사용

```bash
# 출력 디렉토리 지정
python get_ohlcv_incremental.py --output kospi_data

# API 호출 간 지연 시간 조정 (초)
python get_ohlcv_incremental.py --delay 0.2

# 주봉 데이터 조회
python get_ohlcv_incremental.py --period W
```

## 작동 방식

### 첫 실행 (신규 종목)

1. 종목별 CSV 파일이 없으면 1997년부터 데이터를 가져옵니다
2. API 제한을 고려하여 5000개씩 배치로 나눠서 가져옵니다
3. 모든 데이터를 하나의 CSV 파일로 저장합니다

### 재실행 (기존 데이터 있음)

1. 각 종목의 CSV 파일에서 마지막 날짜를 확인합니다
2. 마지막 날짜 이후부터 현재까지의 데이터만 가져옵니다
3. 새 데이터를 기존 데이터와 병합하여 저장합니다
4. 이미 최신 데이터가 있으면 스킵합니다

## 예상 소요 시간

- **첫 실행 (전체 데이터)**: 
  - 종목 수: 약 2,000개
  - 종목당 약 6,500일 (1997~2024)
  - 배치 처리: 종목당 2회 API 호출
  - 예상 시간: 약 7-10시간 (2,000개 × 2회 × 0.1초)

- **재실행 (증분 데이터)**:
  - 종목당 최근 며칠 데이터만
  - 예상 시간: 약 3-5분

## 출력 파일

- **개별 종목**: `ohlcv_data/{종목코드}.csv`
  - 각 종목의 1997년부터 현재까지 전체 데이터
  - 컬럼: 날짜, 시가, 고가, 저가, 종가, 거래량, 거래대금, 종목코드

- **통합 파일**: `ohlcv_data/all_kospi_ohlcv_{날짜}.csv`
  - 모든 종목의 데이터를 하나로 통합

- **실패 종목**: `ohlcv_data/failed_codes_{날짜}.txt`
  - 데이터를 가져오지 못한 종목 목록

## 중단 후 재개

프로그램이 중간에 중단되어도 문제없습니다:

1. 이미 저장된 데이터는 그대로 유지됩니다
2. 다시 실행하면 저장된 데이터 이후부터 이어서 가져옵니다
3. 실패한 종목은 `failed_codes_{날짜}.txt` 파일에 기록됩니다

## 주의사항

1. **첫 실행은 시간이 오래 걸립니다**: 전체 데이터를 가져오려면 7-10시간 정도 소요될 수 있습니다
2. **API 호출 제한**: 기본 지연 시간(0.1초)을 유지하는 것을 권장합니다
3. **디스크 공간**: 약 2,000개 종목 × 6,500일 = 약 1-2GB 정도의 공간이 필요할 수 있습니다
4. **토큰 만료**: 토큰은 24시간마다 자동 갱신됩니다

## 진행 상황 확인

프로그램 실행 중:
- 100개 종목마다 진행 상황이 출력됩니다
- 각 종목의 상태(성공/스킵/실패)가 표시됩니다
- 새로 추가된 데이터 개수가 표시됩니다

## 예시 출력

```
[1/2022] 000020   신규 종목, 1997년부터 데이터 조회 (약 9850일)
  ✓ (5000개 새 데이터 추가)

진행 상황: 100/2022 (4.9%)
성공: 95, 스킵: 3, 실패: 2, 새 데이터: 45000개
```
